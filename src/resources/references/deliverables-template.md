# 成果物とマニュアル テンプレート

**全ペルソナの作業が完了した後、最終成果物として `成果物とマニュアル.md` をプロジェクトルートに生成する。** このファイルはユーザーが開発成果物の全体像を把握し、スクラッチ組織での手動テストをスムーズに進めるためのガイドブック。日本語で記述すること。

ユーザーの指示を待たずに、スクラッチ組織構築者の作業完了後に自動で生成する。

## 5. デプロイ手順（本番環境向け）

スクラッチ組織での検証が完了し、本番環境にデプロイする場合：

1. デプロイ検証（ドライラン）
   ```bash
   sf project deploy validate --source-dir force-app --target-org <本番org-alias>
   ```

2. デプロイ実行
   ```bash
   sf project deploy start --source-dir force-app --target-org <本番org-alias>
   ```

3. 本番環境でテストクラスを実行し、カバレッジを確認
