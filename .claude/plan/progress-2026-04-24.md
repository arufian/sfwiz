## Changes this session (chronological)

### 1. Keybind hint bar → side panel

Removed bottom-of-`InputLine` hint row (`Enter send · Shift+Enter newline · ...`). Added new `KeybindHints` component inside `SidePanel` (bottom). Covers Enter / Shift+Enter / Ctrl+B / Ctrl+H / Ctrl+Q.

Splash now also renders `SidePanel` (was hidden pre-chat) so keys stay visible from launch.

File: `src/poc.tsx` — `KeybindHints`, `SidePanel`, `InputLine` (prop removed), `App` (splash layout).

### 2. Empty-Enter on splash

Hardened guard in `handleSubmit`: if user hits Enter with empty textarea while on splash, splash stays + toast fires `type something first · splash stays until you ask`. Non-splash empty-Enter still silently ignored.

### 3. Focus lock on input

`<scrollbox focused>` removed — chat panel no longer steals focus after demo load. Input textarea remains only focused element (gated by `!modalOpen && !helpOpen`).

### 4. Chat visuals

- Removed `"chat"` text label above chat scrollbox.
- User blocks wrapped in colored row: `backgroundColor: '#0d2a33'` (dark teal) + lateral padding. Chevron + text inherit same bg so fill is contiguous. Assistant/tool/divider blocks untouched → user input visually distinct.

### 5. Data-query demo scenarios

Appended 3 user-driven SF-data scenarios to `INITIAL_CHAT`:

- **Aggregate**: `how many open opps > $50k per AE this quarter?` → `sf_sobject_describe` + `sf_query` (GROUP BY OwnerId) → assistant table (6 AEs, $2.52M total).
- **Raw SOQL**: user pastes full `SELECT Id, Name, StageName, Amount FROM Opportunity WHERE Owner.Name = 'Marcus Chen' AND IsClosed = false` → `sf_query` → 5 rows detail expanded.
- **Top-N aggregate**: `top 3 accounts by open pipeline?` → `sf_query` GROUP BY Account.Name → ranked 3-row summary.

All 3 use existing `ToolBlock`/`ChatBlocks` render paths — no new component.

### 6. Layout crash after Ctrl+D

Bug: after loading 50+ block demo, `InputLine` border collapsed onto placeholder → text appeared as dashes `ask-anything...-"Fix-FLS-on-OppHandler"`.

Cause: flex children all defaulted to `flexShrink: 1`; chat `flexGrow: 1` won layout war and squeezed chrome to 0 rows → top/bottom border chars overlapped content row.

Fix: `flexShrink: 0` added to outer boxes of `StatusBar`, `ToastBar`, and `InputLine`. Chat still absorbs remaining height; chrome heights now pinned.
